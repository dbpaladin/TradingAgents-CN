# A股策略回测使用指南

## 概述

当前 Web 版已经支持 A 股 AI 回测任务。回测会在历史交易日上重复运行分析链路，并按 A 股交易约束模拟买卖结果。

适用入口：

- 前端页面：`A股策略回测`
- 后端接口：`/api/backtest/tasks`

## 当前支持能力

- 按历史日期循环执行 AI 分析
- 支持选择分析师组合
- 支持快速 / 深度两种研究深度
- 支持为回测单独指定快速模型与深度模型
- 支持标准 / 加速 / 极速三种分析频率
- 支持 A 股交易约束：T+1、涨跌停、佣金、印花税、最低佣金
- 支持回测任务列表、进度查询、结果查询、任务删除

## 回测参数说明

### 基础参数

- 股票代码：支持 `000001` 或 `000001.SZ` 这类 A 股代码
- 回测区间：开始日期必须早于结束日期
- 初始资金：默认 `100000`，最低 `10000`
- 买入仓位比例：默认 100%，前端按 10% 步进调整

### AI 参数

- 分析师：建议首轮只选 `market` + `fundamentals`
- 研究深度：`快速` 更省 Token，`深度` 更适合短区间精查
- 快速分析模型：用于轻量分析节点
- 深度决策模型：用于更重的综合判断

### 回测速度模式

- 标准：`decision_interval_days = 1`，每个交易日都重新运行 AI
- 加速：`decision_interval_days = 3`，每 3 个交易日重算一次
- 极速：`decision_interval_days = 5`，每 5 个交易日重算一次

当 `decision_interval_days > 1` 时，中间交易日会复用上一次 AI 信号，用于大区间粗筛。

## 推荐使用方式

### 首轮粗筛

- 研究深度：`快速`
- 分析师：`market` + `fundamentals`
- 速度模式：`加速` 或 `极速`
- 区间：先用 3 到 12 个月

### 二次验证

- 保留候选标的
- 切回 `标准`
- 按需要增加 `fund_flow`、`theme_rotation`、`institutional_theme`
- 缩短到更关注的行情阶段复核

## 结果解读

回测结果会返回三类核心数据：

- `metrics`：总收益率、年化收益率、超额收益、最大回撤、夏普、波动率、胜率、费用等
- `trades`：逐笔交易记录，包括价格、数量、费用、交易后资产、AI 信号摘要
- `daily_equity`：每日净值、现金、持仓市值、回撤、基准净值

如果任务仍在执行中，结果接口会先返回当前状态和进度。

## 成本控制建议

- 优先减少分析师数量，而不是盲目拉长区间
- 优先使用 `快速` 深度做首轮筛查
- 优先使用 `加速` / `极速` 模式筛选大区间
- 只有在候选标的进入复核阶段时再切换到 `标准`

## 常见问题

### 为什么回测速度比单次分析慢很多

因为回测会在多个历史交易日重复调用分析链路，区间越长、分析师越多、模型越重，耗时和成本越高。

### 为什么中间日期没有重新分析

通常是启用了 `加速` 或 `极速` 模式，这是预期行为。

### 为什么某天信号没有实际成交

A 股回测会考虑：

- T+1 限制
- 涨停无法买入
- 跌停无法卖出
- 最低佣金与印花税

因此 AI 给出信号不代表一定能成交。
