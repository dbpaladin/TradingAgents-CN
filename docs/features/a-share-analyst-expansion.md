# A股分析师扩展与联动说明

## 概述

最近几轮 A 股链路补齐了 3 个更偏交易语境的分析师视角，并将它们接入前端、回测、报告导出和多智能体主链路：

- `fund_flow`：资金面分析师
- `theme_rotation`：题材轮动分析师
- `institutional_theme`：机构布局题材分析师

## 新增或补强的分析视角

### 资金面分析师

关注：

- 龙虎榜
- 机构 / 游资风格
- 北向资金
- 融资融券

更适合回答：

- 当前是谁在买
- 是趋势跟随还是分歧博弈
- 资金证据是否支持交易

### 题材轮动分析师

关注：

- 主线题材
- 板块轮动
- 龙头梯队
- 个股题材定位

更适合回答：

- 目标股处于主线核心、外延，还是非主线
- 当前阶段更像启动、加速、分歧还是退潮

### 机构布局题材分析师

关注：

- 机构提前布局的候选题材
- 试盘、酝酿和潜在扩散方向

更适合回答：

- 哪些方向更可能被机构提前布局
- 个股是否具备“先于市场共识”的题材属性

## 主链路联动

这三类报告已经不只是展示模块，而是会参与：

- 研究员汇总
- 交易员投资计划
- 风险经理判断
- 风险辩论
- 反思链路

因此它们会直接影响最终建议，而不是只在报告里“多一段说明”。

## 前后端同步范围

当前已同步到：

- 单股分析入口
- Web 回测入口
- 报告详情页
- 个股详情页
- 导出链路
- Streamlit 展示链路
- 进度提示与分析师标签显示

## 别名行为

为了减少用户理解成本，`sentiment` 在 A 股场景下不再只代表单一情绪维度，而会扩展包含部分相关分析师。

最近补齐后，前端与主链路中与 `sentiment` 相关的别名扩展已经覆盖：

- `fund_flow`
- `theme_rotation`
- `institutional_theme`

实际使用时，建议仍然在页面中显式勾选分析师，以避免对成本和耗时产生误判。

## 对回测的影响

回测页面已支持选择这些分析师，但要注意：

- 分析师越多，单次 AI 重算成本越高
- 在 `加速` / `极速` 模式下更适合先粗筛
- 精查时再缩短区间并切回 `标准`

推荐组合：

- 首轮：`market + fundamentals + fund_flow`
- 强题材股复核：增加 `theme_rotation`
- 机构博弈或中线题材复核：再增加 `institutional_theme`
