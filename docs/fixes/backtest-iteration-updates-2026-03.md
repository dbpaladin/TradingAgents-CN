# A股回测迭代更新记录（2026-03-19 至 2026-03-21）

## 概述

回测功能不是一次性完成的，而是在 2026-03-19 到 2026-03-21 之间连续补了几轮能力。

这份记录用于说明这些提交累计带来的结果，方便后续定位为什么回测页面、任务进度和成本控制看起来比最初版本更完整。

## 第一阶段：基础回测能力落地

最初落地的能力包括：

- A 股回测任务创建
- 回测任务状态查询
- 回测结果查询
- 回测任务删除
- 基础前端页面与侧边栏入口
- MongoDB 持久化任务结果

同时纳入了 A 股交易约束：

- T+1
- 涨停无法买入
- 跌停无法卖出
- 佣金
- 印花税
- 最低佣金

## 第二阶段：回测体验增强

随后几轮主要增强了“能不能更快、更好用、更可观察”：

- 页面增加历史回测列表
- 结果看板和进度展示更完整
- 支持查看当前日期与当前步骤
- 后端返回更细粒度的进度消息
- 任务中心对回测任务的展示更友好

## 第三阶段：速度模式与成本控制

为了降低长区间回测成本，后续增加了按交易日间隔复用 AI 信号的能力：

- `decision_interval_days = 1`：标准模式
- `decision_interval_days = 3`：加速模式
- `decision_interval_days = 5`：极速模式

此外还补了：

- 预计交易日计算
- 预计 AI 重算次数提示
- 根据区间和分析师数量自动优化间隔的逻辑

这让回测从“只能硬跑全量”升级成“可以先粗筛再精查”。

## 第四阶段：分析师与模型配置补齐

回测入口后续还支持了：

- 自定义分析师组合
- `fund_flow` 接入回测
- 独立指定快速模型和深度模型
- 研究深度与速度模式联动使用

这使回测不再只是简单重放，而是能更接近真实分析链路。

## 第五阶段：稳定性与防卡死加固（2026-03-22）

针对“回测长期停留在低进度（例如 12%）”的问题，补充了任务级容错与降级机制：

- 单日 AI 分析增加硬超时，避免某一天无限阻塞
- AI 分析超时后自动重建线程池，规避卡死线程污染后续日期
- 连续超时达到阈值后自动降级为 `HOLD`，优先保证整段回测可完成
- 回测启动阶段提前初始化 AI 运行时并做 API Key 可用性检查
- 当供应商需要 API Key 但缺失时，快速降级为 `HOLD`，避免长时间无效等待

本阶段重点是“先保证能稳定跑完，再优化收益质量”，用于处理模型配置缺失、上游波动或调用链异常导致的长时间假运行。

## 当前推荐用法

### 粗筛

- 分析师：`market + fundamentals`
- 研究深度：`快速`
- 速度模式：`加速` 或 `极速`

### 复核

- 缩短区间
- 切回 `标准`
- 再按需要加入 `fund_flow`、`theme_rotation`、`institutional_theme`
