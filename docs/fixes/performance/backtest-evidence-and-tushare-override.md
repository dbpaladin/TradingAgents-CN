# 回测资金证据与第三方 Tushare 接入修复说明

- 日期：2026-03-24
- 适用范围：A 股回测、资金面分析、数据源管理
- 项目路径：`/home/wing/myproject/TradingAgents-CN`

## 背景

本轮问题集中在三个表象：

1. 最新回测显示 `0交易 / 0回撤`，但并不代表策略天然稳定。
2. 资金面报告中频繁出现“资金证据不完整”，并被下游策略放大为保守甚至偏空结论。
3. 已经约定使用第三方 Tushare endpoint + token，但系统部分链路仍受数据库旧配置影响。

## 根因拆解

### 1. `0交易 / 0回撤` 的主因是 AI 分析超时降级

在线日志表明，回测部分日期在分析阶段触发长时间超时，随后被系统整体降级为 `HOLD`，从而产生“无交易、无回撤”的假象。

性能瓶颈主要不在撮合层，而在 AI 推理链路，尤其是消息清理和新闻相关节点。

### 2. “资金证据不完整”不是单一数据源故障

该提示实际表示资金面关键证据没有同时取全，常见原因包括：

- `moneyflow / margin_detail / hk_hold` 某一项未返回
- 目标股票在当日没有对应披露记录
- 接口可用但股票或日期维度未命中
- 北向或融资融券口径当天本就不适用

因此它更接近“证据覆盖不足”，而不是“程序报错”。

### 3. 第三方 Tushare 规则在 provider 层和 manager 层不一致

`TushareProvider` 已支持：

- 自定义 `TUSHARE_ENDPOINT`
- `.env` 中第三方 endpoint + token 成对优先

但 `DataSourceManager` 之前仍会先读取数据库 `data_source_configs` 中的 `enabled` 标记；如果数据库里 `tushare.enabled=false`，即使 `.env` 已配置第三方地址，也会被提前跳过。

## 修复内容

### 资金面证据解释修复

- 为资金面摘要新增证据完整度分数、标签、缺失项和数据质量说明
- 将“北向未命中”“融资融券缺失”明确标记为数据缺口，而不是默认偏空
- 强化分析师、交易员、风控提示词：数据缺口只能降低置信度，不能单独推出卖出结论

### 回测稳定性修复

- 收紧回测专用运行时约束，减少记忆与多轮辩论带来的超时成本
- 长区间任务自动使用轻量分析组合，降低首日即超时的概率
- 移除“连续空仓后强制买入”的入场保护，回测结果保持真实交易机会分布
- 前端耗时展示改为按 `ms / s / 分钟 / 小时` 自适应换算

### 第三方 Tushare 覆盖修复

- 在 `DataSourceManager` 中新增第三方 endpoint 检测
- 当 `TUSHARE_ENDPOINT` 不是官方地址且 `TUSHARE_TOKEN` 有效时：
  - 可用数据源检测会按 `.env` 启用 Tushare
  - 数据源优先级读取也会保留 Tushare
- 官方默认 endpoint 不触发该覆盖规则，避免误放开数据库禁用项

## 验证结果

- `.venv/bin/pytest -q tests/test_tushare_env_override.py` 通过
- 回归测试覆盖：
  - 第三方 endpoint 可绕过数据库 disabled
  - 官方 endpoint 不可绕过数据库 disabled
- 使用当前 `.env` 实测，`available_sources` 与优先级顺序均已包含 `tushare`

## 结论

这次修复后的系统行为更符合实际预期：

- 没有交易机会时，不再被“硬性买入”污染回测结果
- “资金证据不完整”会被视为证据覆盖不足，而不是隐式利空
- 第三方 Tushare 接入规则在 provider 和 manager 两层已经统一，不会再被数据库中的旧官方配置误伤
