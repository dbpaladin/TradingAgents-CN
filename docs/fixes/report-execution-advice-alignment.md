# 报告方向判断与执行建议对齐修复

## 背景

单股分析报告中同时存在以下几类结论来源：

- 基本面模块里的“投资建议”
- 交易计划里的仓位/风控动作
- 风险经理 / 投资组合经理的最终裁决
- `final_trade_decision.md` 中落盘的结构化三分类动作

在 `002709` 的最新报告复核中，暴露出一个关键问题：

- 最终裁决正文常包含“持仓者减仓观察、空仓者等待确认”这类执行层指令
- 但结构化决策只有 `买入 / 持有 / 卖出` 三分类
- 最终报告和摘要在落盘时只展示三分类，导致“持有但减仓”被压平为普通“持有”

这会让用户误以为“投资建议”和“投资组合经理”结论相反，实际根因是执行语义在结构化阶段丢失。

## 根因

### 1. `SignalProcessor` 只保留方向，不保留执行动作

文件：`tradingagents/graph/signal_processing.py`

- 现有链路会从最终决策文本提取 `action`
- 也会根据目标价、减仓/等待确认等语义修正 `action`
- 但不会把“持仓者怎么做 / 空仓者怎么做”保留下来

### 2. 服务层推荐文案只展示三分类动作

文件：`app/services/simple_analysis_service.py`

- `recommendation` 文案以前直接输出 `投资建议：{action}`
- `final_trade_decision.md` 以前也只写 `行动`
- 结果是复杂执行意见在最终用户界面中被压缩成单标签

## 修复内容

### 1. 新增执行建议提取

文件：`tradingagents/graph/signal_processing.py`

- 新增 `_extract_execution_advice()`
- 优先提取：
  - `持仓者建议：...`
  - `空仓者建议：...`
  - `操作建议 / 执行建议 / 明确建议 / 立刻执行 / 战略行动：...`
- 若正文中没有明确标签，则根据常见语义生成保守兜底：
  - `持有` + `减仓观察/不追涨/等待确认` -> `持仓者减仓观察，空仓者等待确认`
  - `卖出` + `减仓/清仓/分批卖出` -> `分批减仓或卖出，优先控制回撤`
  - `买入` + `回踩/分批买入/小仓试错` -> `等待合适位置后分批买入，避免一次性重仓`

### 2. 决策结果新增 `execution_advice`

文件：`tradingagents/graph/signal_processing.py`

- 在 `process_signal()` 返回值中加入 `execution_advice`
- 在 `_extract_simple_decision()` 和默认返回路径中同样补齐该字段

### 3. 最终推荐文案改成“双层表达”

文件：`app/services/simple_analysis_service.py`

- 以前：
  - `投资建议：持有。`
- 现在：
  - `方向判断：持有。执行建议：持仓者减仓观察；空仓者等待确认。`

这样可以明确区分：

- 方向层：买入 / 持有 / 卖出
- 执行层：加仓 / 减仓 / 观察 / 等待确认 / 分批卖出

### 4. 最终决策报告落盘格式同步调整

文件：`app/services/simple_analysis_service.py`

- 以前字段：
  - `行动`
- 现在字段：
  - `方向判断`
  - `执行建议`

## 测试覆盖

新增 / 更新测试：

- `tests/test_signal_processor_decision_consistency.py`
- `tests/test_simple_analysis_service_report_fallbacks.py`

验证点：

- 当原文存在 `持仓者建议 / 空仓者建议` 时，`execution_advice` 能被保留
- 当原文标题是“持有”但执行层要求明显降仓时，仍可修正为 `卖出`
- 最终 `final_trade_decision.md` 会同时输出 `方向判断` 与 `执行建议`

## 效果

修复后，类似 `002709` 这类报告将不再把：

- “持有，但减仓观察、不追涨、等待确认”

错误展示成：

- “持有”

而会展示为：

- `方向判断：持有`
- `执行建议：持仓者减仓观察；空仓者等待确认`

这能显著降低用户对“投资建议”和“投资组合经理”结论冲突的误读。
