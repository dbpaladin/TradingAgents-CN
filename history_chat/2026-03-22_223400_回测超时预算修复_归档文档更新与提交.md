## 排查结论

这次问题不是交易规则错误，也不是 API Key 缺失，而是回测超时预算低估了完整图执行耗时：

- 任务 `42bbdc1b-5ea5-474a-8934-7b21bfc5716c`
- 标的：`601669`
- 区间：`2026-01-01` 至 `2026-03-22`
- 研究深度：`快速`
- 分析师：`market`、`fundamentals`、`sentiment`、`fund_flow`、`news`、`institutional_theme`、`theme_rotation`
- 决策间隔：`5`

日志显示，虽然模型单次请求超时配置是 `180s`，但完整分析图的真实总耗时接近 `989.61s`。旧逻辑仍将单日回测硬超时上限固定限幅到 `480s`，导致：

- 实际分析尚未结束时就被 `asyncio.wait_for(...)` 打断
- 连续两天命中超时后触发保护降级
- 后续交易日全部被强制记为 `HOLD`

另外，本次任务包含 `sentiment`，图执行时会展开为 `social + emotion` 两个独立分析节点，旧的简单计数也会进一步低估负载。

### 1) 回测超时预算改为动态估算

  - `sentiment -> social + emotion` 的展开成本

对于本次 `601669` 的原始任务配置，新预算已提升到：

这与此前日志中的真实执行耗时量级基本对齐，不会再在 `480s` 被误判超时。
